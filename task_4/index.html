Task 1

First, create a bash script that will install the latest version of Docker. keep in mind, if there is any older version of docker is installed in the system, you have to remove it first. After installing Docker, the script will pull the images of Elasticsearch and Kibana (any version between 7.7 to 7.12).

    Users are advised to determine their sha-bang by typing 
    which bang
    and change accordingly inside the script files in task_1 and task_2 folder

>

    Launch the terminal with administrative privilege

sudo su

    From the terminal, change your directory to task_1 folder using the command cd

cd {directory}

    Type 
     chmod +x task_1.sh
    to make it executable

    Finally type 
     ./task_1.sh

    to install Docker and pull Elasticsearch 7.7.0, Kibana 7.7.0 images

    [Recommended only if task_1.sh fails]

    Some users might experience DNS issue while installing Docker. In that case, it is advised to make some change on top of /etc/resolv.conf file 
    nano /etc/resolv.conf
    and put the below codes respectively. 
    nameserver 8.8.8.8 
                            
    nameserver 8.8.4.4

Task 2

Create another bash script that will run Elasticsearch on port 2048 and Kibana on port 4096 in the background. You should be able to access Kibana at http://localhost:4096

    From the terminal, change directory to task_2 folder just like we previously did it using cd command.

    In order to run the Elasticsearch and the Kibana containers :

    Type 
     chmod +x task_2.sh
    to make it executable

    Finally type 
     ./task_2.sh

    Kibana dashboard may take a couple of seconds upto 1 minute depending on the host machine at localhost:4096

Task 3

Let's say, You need to FAKE some student details for some education purpose. But, you can not create more than 10 student on a five minutes window. You may wonder, "WHY"? Believe me, I don't know either. So, you are going to create a python script that will generate 10 FAKE student details i.e. name, age, gender, address, contact and so on for a student and store that to an index naming students on the very same Elasticsearch you have launched in the first task. Please don't use the childish time.sleep() method in your script. Instead, use some kind of scheduler, and you are free to choose any of your favourite one.

>

    From the terminal, change directory to task_3 folder just like we did before

    Now, its important that we create a virtual environment on the host machine. To do that, we need to type the following command

python3 -m venv venv

    Please note that depending on the configuration of python on the host machine, you might need to change command to [python] instead of [python3]

    Now to activate the virtual environment, Type

source venv/bin/activate

    Once the venv has been activated, we need to install two python libraries named : Elasticsearch and Schedule. please also note that we will use the 7.7.0 version of elasticsearch client, so we need to specify the version along with the command

pip install elasticsearch==7.7.0
pip install schedule

    Once its done, we will now type

python3 task_3.py

    in order to run the script.

    Please note again that, we might need to substitute the [python3] command to [python] depending on the python installed on the host machine.
